# PocketForge 🔨

**Build web apps from your phone — a Replit-style mobile app builder.**

Describe an app in a sentence. Claude designs and codes it, Vercel hosts it,
and the live app appears right inside the iOS app — with an agent chat to
iterate and a code browser to inspect every file.

```
┌─────────────┐   mutations/actions    ┌─────────────────┐
│   iOS app   │ ─────────────────────▶ │     Convex      │
│  (SwiftUI + │ ◀───────────────────── │  db + realtime  │
│ ConvexMobile)│   live subscriptions  │  + Node actions │
└──────┬──────┘                        └────────┬────────┘
       │                                        │
       │ WKWebView loads preview URL            │ Anthropic API (Claude)
       ▼                                        ▼
┌─────────────────────────────────────────────────────────┐
│     Vercel static deployment (one project per app)      │
│     generated app served on its own public URL          │
└─────────────────────────────────────────────────────────┘
```

## How it works

1. **Enter App** — no sign-in; one tap and you're in (single-user by design).
2. **Describe** — "A pomodoro timer with a glowing countdown ring…"
3. **Forge** — a Convex Node action asks Claude (`claude-opus-4-8`) for a
   complete, mobile-first static web app (HTML/CSS/JS, CDN libraries, no
   build step), saves the files to Convex, and publishes them as a Vercel
   static deployment.
4. **Run** — the deployment URL loads in an embedded WKWebView. Build
   progress, chat, and code all stream live over Convex subscriptions.
5. **Iterate** — chat with the agent ("make it dark mode") and a fresh
   deployment goes live in seconds.

Generated apps are deliberately **static-site + CDN** (no npm/build step):
builds are fast, never flaky, and previews load instantly on a phone.
`localStorage` gives them real persistence.

## Setup (≈10 minutes)

### 1. Backend — Convex

```bash
cd pocketforge/convex
npm install
npx convex dev          # creates a free Convex project, deploys, watches
```

The CLI prints your deployment URL (e.g. `https://happy-animal-123.convex.cloud`).

Set the two API keys the agent needs:

```bash
npx convex env set ANTHROPIC_API_KEY sk-ant-...   # console.anthropic.com
npx convex env set VERCEL_TOKEN ...               # vercel.com/account/tokens → Create Token
# Only if your Vercel account is part of a team:
# npx convex env set VERCEL_TEAM_ID team_...
```

For production use `npx convex deploy` instead of `dev`.

### 2. iOS app

1. Open `pocketforge/ios/PocketForge.xcodeproj` in **Xcode 16+**.
2. Xcode resolves the [`convex-swift`](https://github.com/get-convex/convex-swift)
   package automatically on first open.
3. Edit `PocketForge/AppConfig.swift` and paste your Convex deployment URL.
4. Select your signing team (Signing & Capabilities), pick a simulator or
   device, and **Run**.

### 3. Forge something

Tap **Enter App** → **+** → describe an app → **Build**. Watch the status
stream, then open the **App** tab when it flips to **Live**. Use the
**Agent** tab to request changes and the **Code** tab to read the source.

## Notes & knobs

- **Model**: `claude-opus-4-8` with adaptive thinking, streamed (large
  generations won't time out). Change it in `convex/convex/agent.ts`.
- **Hosting**: each app gets its own Vercel project
  (`pocketforge-<name>-<id>`), created automatically on first deploy with
  `framework: null` (plain static hosting — immune to framework
  misdetection). Deployments are immutable and never sleep, so preview and
  share links stay live forever.
- **Deleting an app** (menu in the workspace) deletes its Vercel project too.
- **No auth** is intentional per the product design: a single implicit user.
  Don't ship public TestFlight builds without adding Convex Auth first —
  anyone with the deployment URL could call the functions.
- `convex/_generated/` is checked in so the backend typechecks out of the
  box; `npx convex dev` keeps it up to date automatically.

## Troubleshooting

| Symptom | Fix |
|---|---|
| App shows no projects / create fails | `AppConfig.swift` URL wrong, or backend not deployed (`npx convex dev` running?) |
| Build fails instantly with env error | Set `ANTHROPIC_API_KEY` / `VERCEL_TOKEN` via `npx convex env set` |
| Vercel deploy fails with 403 | Token expired or wrong scope — create a new token; set `VERCEL_TEAM_ID` if the account is in a team |
| Xcode can't resolve packages | Xcode → File → Packages → Reset Package Caches |
